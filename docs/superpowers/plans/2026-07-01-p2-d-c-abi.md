# P2-D C-ABI Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Publish a stable C-ABI on top of `e2v-api` with opaque handles, stable error codes, owned return buffers, panic-safe exports, a public header, and consumable ABI tests.

**Architecture:** Extend `e2v-api` with a dedicated `c_abi` module that wraps the stable Rust SDK instead of exposing internals. Represent Rust-owned objects as opaque handles, map `SdkErrorCode` into a `#[repr(C)]` ABI enum, serialize complex outputs to owned JSON strings, and guard every exported function with a shared `catch_unwind` wrapper.

**Tech Stack:** Rust workspace, `e2v-api`, `serde_json`, `std::panic::catch_unwind`, `cdylib` crate output, C header generation, Rust integration tests, and a compiled C smoke test.

```rust
#[repr(C)]
#[derive(Debug, Clone, Copy, PartialEq, Eq)]
pub enum e2v_error_code_t {
    E2V_OK = 0,
    E2V_INVALID_ARGUMENT = 1,
    E2V_NOT_FOUND = 2,
    E2V_INTERNAL_PANIC = 255,
}

fn ffi_call<F>(error_out: *mut *mut e2v_error_t, f: F) -> e2v_error_code_t
where
    F: FnOnce() -> crate::SdkResult<()> + std::panic::UnwindSafe,
{
    match std::panic::catch_unwind(f) {
        Ok(Ok(())) => e2v_error_code_t::E2V_OK,
        Ok(Err(err)) => store_error(error_out, err),
        Err(_) => store_panic(error_out),
    }
}
```
